# DESCRIPTION OF TASK

This project demonstrates Data-Driven Testing using Selenium WebDriver and Python. The test data is stored in a CSV file, allowing the same test case to be executed with multiple sets of input data without changing the code.

The automation script reads usernames and passwords from the CSV file, opens the target website, performs login operations for each data set, and verifies the results. The execution status (PASS/FAIL) is recorded in a results file for analysis.

This approach improves test coverage, reduces manual effort, and makes it easy to test multiple scenarios using a single automation script.

# Technologies Used

# Features

* Reads test data from a CSV file
* Executes automated login tests
* Supports multiple input combinations
* Logs test execution results
* Reduces repetitive manual testing

# Deliverable

A Selenium automation script that executes tests using multiple input sets from a CSV file and stores the test results in a log file.
